# 학술제 수상 준비도 평가

> 기준: 최신 v2 결과 `output/multi_model_results_v2.json`
> 갱신일: 2026-07-03

## 한 줄 결론

프로젝트는 이전보다 훨씬 좋아졌다. 48개 시나리오, 4개 모델, 4개 조건, 실제 멀티턴 도구 호출 로그까지 갖췄고, “모델에게 조심하라고 말하는 것보다 인터페이스에서 민감 필드를 제거해야 한다”는 메시지도 선명하다.

다만 지금은 “객관적 보안 검증”보다는 **인터페이스 최소권한 정책의 MVP형 실험 연구**로 포지셔닝하는 것이 안전하다. 특히 민감도 가중치, LLM-as-judge 채점, 공격 payload 위치 가정은 명확히 한계로 밝혀야 한다.

## 현재 준비도

| 심사 관점 | 평가 | 근거 |
|---|---:|---|
| 대주제 적합성 | 9/10 | AI 에이전트, 개인정보 자기결정권, 최소권한 설계가 주제와 직접 연결 |
| 시의성 | 9/10 | 도구 사용형 AI와 프롬프트 인젝션은 최신 이슈 |
| MVP/시연 가능성 | 8/10 | 주소록·메일·캘린더, 접근 로그, 필드 필터 차이를 화면으로 보여주기 쉬움 |
| 연구 질문 선명도 | 8/10 | “AI가 어디까지 읽어야 하는가?”가 직관적 |
| 실험 구현 | 8/10 | 실제 멀티턴 루프, 48시나리오, 4모델, 4조건 |
| 데이터 완성도 | 7/10 | 768개 deterministic run은 균형적이나 seed 반복은 없음 |
| 통계적 설득력 | 6/10 | paired McNemar는 적절하지만 효과 없음이 아니라 “검출 안 됨”으로 해석해야 함 |
| 위험 점수 객관성 | 5/10 | 가중치 기반 지표는 유용하지만 외부 근거·민감도 분석 설명이 필요 |
| 공격 실증성 | 5/10 | email body payload 차단은 보이나 subject/calendar/contact payload는 미검증 |
| 재현성 | 7/10 | aggregate JSON fallback을 추가했으나 raw JSONL 원본은 커밋되어 있지 않음 |

**종합: 현재 7/10 전후.**
장려상~우수상 경쟁력은 생겼지만, 대상급으로 보이려면 채점 검증과 공격 시나리오 확장이 필요하다.

## 현재 결과의 강점

1. 필드 필터 없는 A/B와 필드 최소권한 C/D를 명확히 분리했다.
2. 조건 B가 프롬프트만으로는 설계상 노출 용량을 줄이지 못한다는 점을 보여준다.
3. C/D는 합성 데이터와 현재 가중치 기준에서 민감필드 노출 용량을 151.5에서 10.5로 낮춘다.
4. 광범위 업무에서 실현 민감노출이 A 5.31, B 3.59, C/D 0.00으로 갈라진다.
5. 여러 가중치 스킴에서도 C/D가 email body·contact phone/notes 노출을 제거한다는 방향은 유지된다.

## 꼭 조심해야 할 해석

### 1. “업무 비용이 없다”가 아니다

McNemar 결과에서 조건 간 성공률 차이가 유의하지 않게 나온 것은 맞다. 하지만 이는 “비용이 없다”가 아니라 “현재 표본과 채점 방식에서 유의한 비용을 검출하지 못했다”가 정확하다.

### 2. “모든 프롬프트 인젝션을 막았다”가 아니다

현재 악성 지시는 이메일 `body`에 있다. C/D가 `body`를 제거하기 때문에 해당 경로는 차단된다. 그러나 subject, sender, calendar event, contact notes에 payload가 들어가는 경우는 아직 테스트하지 않았다.

### 3. 가중치 점수는 객관적 위험값이 아니다

`contact_phone_notes=5`, `email_body=2`, `calendar_events=1.5`는 명시적 연구자 정의다. 따라서 151.5→10.5 같은 숫자는 “정의한 지표 기준”이지, 보편적 개인정보 위험 점수는 아니다.

### 4. 낮은 성공률의 원인이 아직 분리되지 않았다

성공률은 A 0.25, B 0.24, C 0.20, D 0.23 수준이다. 이게 모델 능력 한계인지, 시나리오 기준이 엄격한지, LLM judge 문제인지는 샘플 사람 평가가 필요하다.

## 수상 가능성 추정

현재 상태 그대로:

- 본선/발표 경쟁력: 75%+
- 장려상 이상: 50~65%
- 우수상 이상: 25~35%
- 대상/최우수상: 8~12%

다음 보강까지 하면:

- 장려상 이상: 65~80%
- 우수상 이상: 40~50%
- 대상/최우수상: 15% 내외

## 우선 보강할 것

1. README에서 “가중치 기반 지표”임을 계속 명시한다.
2. LLM judge 결과 30~50개를 사람이 샘플 검수해 일치율을 제시한다.
3. 악성 payload 위치를 email body 외에 subject/calendar/contact notes로 확장한다.
4. coarse/read_all v3 실험을 완주한 뒤, 현재 granular 결과와 분리해서 보고한다.
5. 라이브 데모는 좌우 비교가 좋다.
   - 왼쪽: A/B, 민감 필드 그대로 전달
   - 오른쪽: C/D, body·phone·notes 제거
   - UI/문서 스타일 후보: [VoltAgent/awesome-design-md](https://github.com/VoltAgent/awesome-design-md)

## 추천 발표 포지셔닝

> 본 연구는 AI 에이전트가 업무 수행 중 접근하는 개인정보 범위와 민감필드 노출을 run 단위로 기록하고, 프롬프트 지시와 인터페이스 최소권한 정책이 실현 민감노출과 업무 성공률에 미치는 영향을 비교한 MVP형 실험이다.

이 포지션이 가장 안전하고 강하다. “완벽한 보안 검증”보다 “측정 가능한 실험 환경과 정책 효과의 구조적 시연”으로 밀어야 한다.
